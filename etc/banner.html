CSV Data Validator
The Kohesio data specification is currently in development. As such the current service is offered as a beta version.

This service allows you validate your data against the CSV data import format expected by Kohesio. Access to this service is anonymous and no data, nor validation reports, are retained following its use. The service is also offered via SOAP API to facilitate machine-to-machine integration.

The input is expected to be provided in CSV format as defined in RFC 4180. Specifically:

  • Fields: A header row is expected to define the input's fields. See the Kohesio data import schema for the expected fields and their data types.
  • Comma character: The expected field delimiter is the comma character (,) although this can be set in the CSV syntax settings (see below).
  • Quote character: Each field header and value may optionally be encapsulated using the double-quote character (").
